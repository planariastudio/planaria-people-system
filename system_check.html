System check

Runs a real, end-to-end pass through KPI, Peer Appraisal, and PIP — creating a real case in Supabase and a real task in ClickUp for each, using real roster identities. This is the most reliable way to confirm the whole chain actually works, because it's the same code path the live forms use — nothing is mocked.
Admin only · unlisted
Before you run this — read this once. The people you pick below will get real ClickUp tasks assigned to them, which usually means a real ClickUp notification. This is not a sandbox.
  • Pick people who won't be confused by a "PIP" or "KPI" task briefly appearing on their name — yourself, if you're on the roster, is the safest choice for every role below.
  • Each run auto-picks a fresh test quarter (Q4 2090–2099) — deliberately outside any real cycle, so cases are easy to spot, never mix with real data, and never collide with a previous run's leftovers.
  • Every note/evidence field this tool writes is prefixed "[SYSTEM CHECK]" so it's unmistakable in ClickUp and Supabase.
  • Use Clean up at the bottom when you're done — it deletes everything this run created from Supabase, ClickUp, and the Sheet, the same cascade the Remove buttons in links_admin.html use.
